# 이미지

> 이미지는 텍스트보다 빠르고 직관적으로 정보를 전달할 수 있으며, 사용자의 이해를 돕습니다.

## 이미지 공통 규칙

1. 비주얼 또는 썸네일 이미지의 포맷은 jpg를 사용 합니다. (png 포맷 사용시 용량 부담)
2. 아이콘 이미지는 svg 또는 png를 사용 합니다. (gif 사용은 되도록 지양 합니다.)
3. 비주얼 또는 썸네일 jpg 이미지의 용량은 500k를 넘지 않도록 퀄리티 조정을 합니다. (용량 문제시 프로젝트 PL과 상의 하기)
4. 반응형 프로젝트를 진행 할 때는 이미지 사이즈를 2배수로 사용 합니다. (상황에 따라 프로젝트 PL과 상의 하여 3배수도 고려 할 수 있습니다.)
5. svg 포맷은 2배수, 3배수를 사용할 필요가 없습니다.

## 이미지 포맷

### 지원되는 이미지 포맷

**1. JPEG (Joint Photographic Experts Group)**

- **특징:** 손실 압축 방식으로 이미지 파일 크기를 크게 줄일 수 있습니다.
- **용도:** 사진 및 복잡한 색상 패턴이 있는 이미지에 적합합니다.
- **장점:** 파일 크기가 작아 로딩 속도가 빠릅니다.
- **단점:** 반복 편집 시 품질 저하가 발생할 수 있습니다.

**2. PNG (Portable Network Graphics)**

- **특징:** 무손실 압축 방식으로 높은 품질을 유지합니다.
- **용도:** 투명 배경이 필요한 이미지나 그래픽, 아이콘에 적합합니다.
- **장점:** 투명도와 반투명도를 지원합니다.
- **단점:** 파일 크기가 JPEG보다 크며, 로딩 시간이 길어질 수 있습니다.

**3. SVG (Scalable Vector Graphics)**

- **특징:** 벡터 형식으로 크기 조절에 따라 품질 저하가 발생하지 않습니다.
- **용도:** 로고, 아이콘, 일러스트 등 크기를 자유롭게 변경할 수 있는 이미지에 적합합니다.
- **장점:** 파일 크기가 작고, 해상도에 독립적입니다.
- **단점:** 복잡한 이미지를 표현하는 데 한계가 있을 수 있습니다.

**4. WebP**

- **특징:** 구글에서 개발한 이미지 포맷으로 손실 압축과 무손실 압축을 모두 지원합니다.
- **용도:** 웹 페이지의 전반적인 성능 향상을 위해 사용됩니다.
- **장점:** 높은 압축률과 품질을 동시에 제공합니다.
- **단점:** 모든 브라우저에서 완벽히 지원되지 않을 수 있습니다.

### 포맷 선택 기준: 용도에 따른 적절한 포맷 선택 가이드라인

**1. 사진 및 복잡한 이미지**

- **포맷 추천:** JPEG
- **선택 이유:** 높은 압축률로 파일 크기를 줄일 수 있으며, 다양한 색상과 복잡한 디테일을 잘 표현합니다.
- **예시:** 웹사이트 배너 이미지, 포토 갤러리

**2. 투명 배경이 필요한 이미지**

- **포맷 추천:** PNG
- **선택 이유:** 투명도 및 반투명도를 지원하며, 고품질의 무손실 압축을 제공합니다.
- **예시:** 로고, 아이콘, 그래픽 요소

**3. 크기 조절이 필요한 벡터 그래픽**

- **포맷 추천:** SVG
- **선택 이유:** 벡터 형식으로 크기 조절에 따른 품질 저하가 없으며, 해상도에 독립적입니다.
- **예시:** 웹사이트 로고, 버튼 아이콘, 일러스트레이션

**4. 웹 페이지 성능 최적화**

- **포맷 추천:** WebP
- **선택 이유:** 높은 압축률로 빠른 로딩 속도를 제공하며, 손실 및 무손실 압축을 모두 지원합니다.
- **예시:** 배너 이미지, 배경 이미지, 상품 이미지

**5. 브라우저 호환성 고려**

- **포맷 추천:** JPEG, PNG (WebP는 대체 포맷으로 준비)
- **선택 이유:** 모든 브라우저에서 호환이 보장되며, 필요 시 WebP를 대체 포맷으로 제공하여 성능을 최적화할 수 있습니다.
- **예시:** 기본 콘텐츠 이미지, 대체 이미지를 제공하는 경우

#### 이미지 포맷 선택 시 유의사항

- **품질과 용량의 균형:** 각 포맷의 특성을 이해하고, 품질과 파일 크기의 균형을 맞춥니다.
- **브라우저 호환성:** 사용 중인 이미지 포맷이 주요 브라우저에서 잘 작동하는지 확인합니다.
- **유지보수:** 이미지 포맷을 결정할 때, 향후 유지보수와 편집의 용이성을 고려합니다.

위 가이드라인을 참고하여 이미지 포맷을 선택하면, 웹 및 모바일 퍼블리싱 작업에서 효율적이고 최적화된 이미지를 사용할 수 있습니다.

## 이미지 최적화

### 이미지 압축 및 크기 조절

#### 이미지 압축

- **손실 압축:** JPEG와 WebP 같은 포맷이 사용됩니다. 품질을 약간 희생하면서 파일 크기를 크게 줄일 수 있습니다.
  - **JPEG 압축:** Photoshop, TinyJPG, JPEG-Optimizer 등 도구를 사용해 압축 비율을 조절할 수 있습니다.
  - **WebP 압축:** WebP Converter, Squoosh 등을 사용해 높은 압축률을 유지하면서도 품질을 관리할 수 있습니다.
- **무손실 압축:** 이미지 품질을 유지하면서 파일 크기를 줄입니다. PNG, GIF, 무손실 WebP 등이 사용됩니다.
  - **PNG 압축:** PNGGauntlet, TinyPNG, OptiPNG 같은 도구를 사용하여 압축합니다.
  - **GIF 압축:** GIF Compressor, Ezgif 등을 사용하여 애니메이션 GIF의 용량을 줄입니다.

**이미지 크기 조절**

- **반응형 이미지:** 다양한 디바이스와 해상도에 맞춰 이미지를 자동으로 조절합니다.
  - **CSS 미디어 쿼리:** `@media`를 사용하여 디바이스 크기에 따라 이미지를 조절합니다.
  - **HTML srcset 속성:** `<img>` 태그에서 `srcset` 속성을 사용하여 다양한 해상도의 이미지를 제공할 수 있습니다.

  ```html
  <img src="small.jpg" srcset="small.jpg 480w, medium.jpg 768w, large.jpg 1024w" alt="Example Image" />
  ```

- **이미지 툴 사용:** Photoshop, GIMP, Sketch 등 이미지 편집 도구를 사용하여 적절한 크기로 조절합니다.
- **자동화 도구:** Gulp, Grunt 등의 빌드 도구를 사용해 이미지 크기 조절을 자동화할 수 있습니다.

### 무손실 압축 도구 소개 및 사용 방법

**1. TinyPNG**

- **소개:** PNG 및 JPEG 파일을 무손실 압축하여 파일 크기를 줄여주는 웹 기반 도구입니다.
- **사용 방법:** TinyPNG 웹사이트(tinypng.com)에 이미지를 업로드하면 자동으로 압축된 이미지를 다운로드할 수 있습니다.

**2. ImageOptim**

- **소개:** Mac 사용자용 이미지 최적화 도구로, 무손실 압축을 통해 파일 크기를 줄입니다.
- **사용 방법:** ImageOptim을 설치하고, 압축할 이미지를 드래그 앤 드롭하면 자동으로 최적화됩니다.

**3. PNGGauntlet**

- **소개:** Windows 사용자용 PNG 파일 무손실 압축 도구입니다.
- **사용 방법:** PNGGauntlet을 설치하고, 압축할 PNG 파일을 추가하여 압축을 시작합니다.

**4. OptiPNG**

- **소개:** PNG 파일의 무손실 압축을 지원하는 오픈 소스 도구입니다.
- **사용 방법:** 명령어 인터페이스를 통해 **`optipng filename.png`** 명령어로 압축을 실행합니다.

**5. Squoosh**

- **소개:** Google의 웹 기반 이미지 최적화 도구로, 다양한 압축 옵션을 제공합니다.
- **사용 방법:** Squoosh 웹사이트(squoosh.app)에 이미지를 업로드하고, 압축 설정을 조절한 후 최적화된 이미지를 다운로드합니다.

### WebP 포맷 활용 방안

**1. WebP 포맷 소개**

- **장점:** JPEG보다 더 높은 압축률을 제공하며, PNG와 같은 무손실 압축도 지원합니다.
- **단점:** 일부 오래된 브라우저에서는 지원하지 않습니다.

**2. WebP 변환 도구**

- **WebP Converter:** 다양한 이미지 포맷을 WebP로 변환할 수 있는 도구입니다. GUI 및 CLI 버전이 있습니다.
- **Squoosh:** Google의 웹 기반 도구로, 이미지를 WebP로 변환할 수 있습니다.
- **Imagemin:** Gulp나 Grunt와 같은 빌드 도구에서 사용할 수 있는 이미지 최적화 플러그인으로, WebP 변환을 지원합니다.

**3. WebP 브라우저 지원**

- **Modernizr 사용:** 브라우저에서 WebP 지원 여부를 감지하여 대응합니다.

  ```jsx
  Modernizr.on("webp", function (result) {
    if (result) {
      // WebP 지원됨
    } else {
      // WebP 지원되지 않음
    }
  });
  ```

**4. HTML에서 WebP 활용**

- **`<picture>` 태그 사용:** 브라우저 호환성을 위해 대체 이미지를 제공하는 방법입니다.

  ```html
  <picture>
    <source srcset="image.webp" type="image/webp" />
    <source srcset="image.jpg" type="image/jpeg" />
    <img src="image.jpg" alt="Example Image" />
  </picture>
  ```
